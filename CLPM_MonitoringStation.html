Apache Superset — Setup & SQL Queries
📦
1 · Connect Database
Add the SQLite database (clpm.db) as a datasource in Superset via Settings → Database Connections → SQLite.
sqlite:////path/to/clpm.db
📊
2 · Scorecard Chart
Use Big Number with Trendline chart type on clpm_metrics table. Filter by controller.
SELECT controller, AVG(aae) as avg_aae, AVG(co_travel) as avg_co_travel, AVG(pct_auto) as pct_auto FROM clpm_metrics GROUP BY controller
📈
3 · Recipe Trend Chart
Line chart on clpm_metrics filtered to PIC-002. Use recipe as color dimension for condition-based limits.
SELECT timestamp, recipe, aae, co_travel, pv_std FROM clpm_metrics WHERE controller = 'PIC-002' ORDER BY timestamp
🎯
4 · Limit Breach Detection
Join clpm_metrics with recipe_limits. Flag when AAE exceeds recipe-specific limit. Use for Value Search equivalent.
SELECT m.timestamp, m.controller, m.recipe, m.aae, l.aae_limit, CASE WHEN m.aae > l.aae_limit THEN 'BREACH' ELSE 'OK' END as status FROM clpm_metrics m JOIN recipe_limits l ON m.controller=l.controller AND m.recipe=l.recipe
⚠️
5 · Mode Monitoring
Mixed Bar + Line chart. Show % time in Auto/Cascade mode over time. Alert when pct_auto drops below 95%.
SELECT DATE(timestamp) as day, controller, AVG(pct_auto) as pct_auto, dominant_mode FROM clpm_metrics GROUP BY day, controller HAVING pct_auto < 95 ORDER BY day
🔄
6 · CO Travel Heatmap
Use Heatmap chart with controller on Y-axis, date on X-axis, co_travel as value. Spot valve wear periods instantly.
SELECT DATE(timestamp) as day, controller, AVG(co_travel) as daily_co_travel FROM clpm_metrics GROUP BY day, controller ORDER BY day, controller
NOTE: Export files available: clpm_metrics.csv, recipe_limits.csv, clpm.db — Upload CSVs directly to Superset as datasets if SQLite connection is unavailable. Current off-the-shelf CLPM tools lack recipe/mode-based condition analysis — this station fills that gap.
